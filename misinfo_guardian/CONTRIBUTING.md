# Contributing to Misinfo Guardian

Thank you for your interest in contributing to Misinfo Guardian!

## Development Workflow

This project is organized into three main components, each managed by a different contributor:

### Component Ownership

- **Extension** (`extension/`): Chrome Extension development
- **Backend** (`backend/`): FastAPI backend and agent logic
- **Infrastructure** (`infra/`): Search tools, MCP integration, and infrastructure

## Getting Started

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/amazing-feature`)
3. Make your changes
4. Commit your changes (`git commit -m 'Add some amazing feature'`)
5. Push to the branch (`git push origin feature/amazing-feature`)
6. Open a Pull Request

## Code Style

- Follow PEP 8 for Python code
- Use meaningful variable and function names
- Add comments for complex logic
- Write docstrings for functions and classes

## Testing

- Write tests for new features
- Ensure all tests pass before submitting a PR
- Add integration tests when appropriate

## Commit Messages

- Use clear, descriptive commit messages
- Reference issue numbers when applicable
- Follow conventional commit format when possible

## Questions?

Feel free to open an issue for any questions or concerns.
